# DayFlow Bot

Локальный Telegram-бот для личного планирования: просмотр календаря, поиск свободных слотов, создание событий и задач обычным языком.

## Что уже есть

- `/today YYYY-MM-DD` показывает события на день
- `/slots YYYY-MM-DD [minutes]` показывает свободные окна
- `/add YYYY-MM-DD HH:MM | Название | [minutes]` создает событие в Google Calendar
- обычные сообщения разбираются через OpenRouter API и превращаются в действие
- бот умеет создавать задачи в Google Tasks
- перед созданием события или задачи бот присылает черновик на подтверждение
- для событий можно хранить свои группы с цветами Google Calendar
- при пересечении событий бот показывает конфликт и дает решить, оставить как есть или удалить конфликтующее событие
- бот умеет редактировать и удалять события
- бот умеет изменять, удалять и завершать задачи

## Важный момент про ChatGPT

Если у вас в самом ChatGPT уже подключен Google Calendar, это не дает Telegram-боту доступ автоматически. Для бота все равно нужен отдельный API-доступ:

- Google Calendar и Google Tasks подключаются через OAuth и файл `credentials.json`
- понимание естественного языка подключается через `OPENROUTER_API_KEY`

## Что нужно

- Python 3.12+
- токен бота от `@BotFather`
- Google Cloud OAuth credentials для Desktop app
- `OPENROUTER_API_KEY` рекомендован для свободных текстовых запросов

## Подготовка Google API

1. Откройте Google Cloud Console.
2. Создайте проект или используйте существующий.
3. Включите `Google Calendar API`.
4. Включите `Google Tasks API`.
5. Создайте `OAuth client ID` типа `Desktop app`.
6. Скачайте JSON и положите его в корень проекта как `credentials.json`.

`token.json` можно использовать только для старого однопользовательского режима. Для Telegram-пользователей теперь используются отдельные токены в `data/google_tokens`.

## Запуск

```powershell
cd D:\dayflow-bot
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
Copy-Item .env.example .env
```

Дальше заполните `.env` и запустите:

```powershell
python bot.py
```

Для OpenRouter обычно достаточно указать:

```env
OPENROUTER_API_KEY=your_key_here
OPENROUTER_MODEL=openai/gpt-4o-mini
```

Если `OPENROUTER_API_KEY` не задан, бот по-прежнему может использовать старый путь через `GEMINI_API_KEY`.

После запуска каждый пользователь Telegram должен сам выполнить:

```text
/connect_google
```

Бот пришлет ссылку Google OAuth. После входа в Google нужно скопировать полный URL из адресной строки, который начинается с `http://localhost:...`, и отправить его обратно боту одним сообщением.

## Примеры команд

```text
/today 2026-03-13
/slots 2026-03-13 30
/connect_google
/add 2026-03-13 15:00 | Созвон с командой | 45
/ask Какие окна лучше оставить под глубокую работу завтра?
```

## Примеры обычных сообщений

```text
Что у меня в календаре завтра?
Найди свободное окно в пятницу на 90 минут
Поставь тренировку 2026-03-14 в 19:00 на час
Добавь задачу оплатить интернет завтра
Создай задачу купить подарок и напиши в заметке позвонить в магазин
Поставь встречу с клиентом 2026-04-01 в 13:00
Создай задачу в списке Работа: подготовить отчет, описание: собрать цифры, подзадачи: написать черновик, проверить данные
Перенеси встречу с клиентом на 2026-04-01 15:00
Удали тренировку завтра
Переименуй задачу отправить отчет в отправить квартальный отчет
Отметь задачу оплатить интернет выполненной
```

## Группы событий

Группы хранятся в локальном файле `data/event_groups.json`.

Команды:

```text
/groups
/group_add Работа | 9
/group_add Личное | зеленый
/group_delete Личное
```

При создании события ИИ пытается подобрать группу. Перед созданием бот покажет черновик, а группу можно сменить кнопкой или снять совсем через `Без группы`.

## Задачи

Бот использует ваши реальные списки Google Tasks. Посмотреть их можно так:

```text
/tasklists
```

ИИ может вытащить из сообщения:

- список задач
- описание
- срок
- подзадачи

- ограничить доступ к боту только вашему Telegram user id
